What\'92s a Web Cache? Why do people use them?\
\pard\tx1660\tx2160\pardeftab720\li2160\fi-2160\partightenfactor0
\ls14\ilvl2\cf4 \kerning1\expnd0\expndtw0 {\listtext	
\f1 \uc0\u8259 
\f0 	}\expnd0\expndtw0\kerning0
A Web cache sits between one or more Web servers (also known as origin servers) and a client or many clients, and watches requests come by, saving copies of the responses \'97 like HTML pages, images and files (collectively known as representations) \'97 for itself. Then, if there is another request for the same URL, it can use the response that it has, instead of asking the origin server for it again.\
\pard\tx566\tx1133\tx1700\pardeftab720\partightenfactor0
\cf4 \
\pard\tx220\tx720\tx1133\tx1700\pardeftab720\li720\fi-720\partightenfactor0

There are two main reasons that Web caches are used:\
\pard\tx940\tx1440\tx1700\pardeftab720\li1440\fi-1440\partightenfactor0
\ls16\ilvl1\cf4 \kerning1\expnd0\expndtw0 {\listtext	
\f1 \uc0\u8259 
\f0 	}\expnd0\expndtw0\kerning0
To reduce latency \'97 Because the request is satisfied from the cache (which is closer to the client) instead of the origin server, it takes less time for it to get the representation and display it. This makes the Web seem more responsive.\
\pard\tx940\tx1440\tx1700\pardeftab720\li1440\fi-1440\partightenfactor0
\ls17\ilvl1\cf4 \kerning1\expnd0\expndtw0 {\listtext	
\f1 \uc0\u8259 
\f0 	}\expnd0\expndtw0\kerning0
To reduce network traffic \'97 Because representations are reused, it reduces the amount of bandwidth used by a client. This saves money if the client is paying for traffic, and keeps their bandwidth requirements lower and more manageable.\

If you examine the preferences dialog of any modern Web browser (like Internet Explorer, Safari or Mozilla), you\'92ll probably notice a \'93cache\'94 setting. This lets you set aside a section of your computer\'92s hard disk to store representations that you\'92ve seen, just for you. The browser cache works according to fairly simple rules. It will check to make sure that the representations are fresh, usually once a session (that is, the once in the current invocation of the browser).\

Proxy Caches\
\pard\tx940\tx1440\tx1700\pardeftab720\li1440\fi-1440\partightenfactor0
\ls19\ilvl1\cf4 \kerning1\expnd0\expndtw0 {\listtext	
\f1 \uc0\u8259 
\f0 	}\expnd0\expndtw0\kerning0
Web proxy caches work on the same principle, but a much larger scale. Proxies serve hundreds or thousands of users in the same way; large corporations and ISPs often set them up on their firewalls, or as standalone devices (also known as intermediaries).\
\ls19\ilvl1\kerning1\expnd0\expndtw0 		\expnd0\expndtw0\kerning0
Because proxy caches aren\'92t part of the client or the origin server, but instead are out on the network, requests have to be routed to them somehow. One way to do this is to use your browser\'92s proxy setting to manually tell it what proxy to use; another is using interception. Interception proxies have Web requests redirected to them by the underlying network itself, so that clients don\'92t need to be configured for them, or even know about them.\
\ls19\ilvl1\kerning1\expnd0\expndtw0 		\expnd0\expndtw0\kerning0
Proxy caches are a type of shared cache; rather than just having one person using them, they usually have a large number of users, and because of this they are very good at reducing latency and network traffic. That\'92s because popular representations are reused a number of times.\
